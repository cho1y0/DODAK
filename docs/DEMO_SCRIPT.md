## STEP 1. 프로젝트 소개 (구두 설명)

> **[화면: 프레젠테이션 슬라이드 또는 발표 자료]**

### 말할 내용

> "안녕하세요. 저희 팀이 개발한 **Dodak(도닥)** 프로젝트를 소개하겠습니다."
>
> "Dodak은 **AI 기반 감정 일기 & 심리 케어 플랫폼**입니다. '도닥'이라는 이름은 사용자의 마음을 따뜻하게 '도닥여준다'는 의미를 담고 있습니다."
>
> "현대인의 감정 방치 문제와 증가하는 우울증·불안장애에 대응하기 위해, 일기를 통한 감정 기록과 **KoBERT 기반 8가지 감정 자동 분석**, **KoGPT2 챗봇 상담**, **감정 맞춤 음악 추천** 기능을 제공합니다."
>
> "기술적으로는 **Spring Boot + FastAPI + MySQL** 3-Tier 아키텍처이며, 의사-환자 연동 시스템까지 포함되어 있습니다."
>
> "그럼 실제 동작하는 시스템을 시연하겠습니다."

> **[화면: /signup]**

### 보여줄 화면
- 단계별 회원가입 폼 (기본 정보, 연락 정보, 주소, 약관 동의)
- 실시간 입력 검증

### 말할 내용

> "회원가입 화면입니다. 프로필 이미지, 가입 역할(일반회원/의사), 아이디, 이름, 비밀번호 등을 입력합니다."
>
> "아이디 중복 체크, 이메일 형식 검증, 비밀번호 규칙(8~20자, 영문+숫자+특수문자) 등 **실시간 유효성 검증**이 적용되어 있습니다."
>
> "우편번호 검색 기능과 약관 동의도 포함되어 있습니다."

### 조작
1. 각 필드 간략히 보여주기
2. 역할 선택 (일반회원/의사) 드롭다운 보여주기
3. (시연용 미리 만든 계정 사용으로 실제 가입은 스킵)

> **[화면: /member/diary/write]**

### 보여줄 화면
- 일기 작성 폼 (날짜, 감정 선택, 제목, 내용)
- 9가지 감정 칩 버튼
- AI 감정 분석 결과

### 말할 내용

> "일기 작성 페이지입니다. 오늘 날짜가 자동 표시되고, 9가지 감정 중 현재 기분을 선택할 수 있습니다."
>
> "제목과 내용을 입력하고 저장하면, **FastAPI 서버의 KoBERT 모델이 자동으로 감정을 분석**합니다."
>
> *(일기 작성 및 저장 후)*
>
> "저장과 동시에 AI가 8가지 감정 비율과 우울증 위험도를 분석했습니다. 기쁨 25%, 슬픔 20%, 불안 10% 등의 결과가 표시됩니다."
>
> "이것이 Dodak의 핵심 기능인 **AI 기반 자동 감정 분석**입니다."

### 조작
1. 감정 칩 선택 (예: 슬픔)
2. 제목 입력 (예: "오늘 하루 힘들었던 일")
3. 내용 입력 (감정이 담긴 텍스트)
4. **"일기 저장하기"** 클릭
5. AI 감정 분석 결과 확인
6. **"기분에 따른 추천 영상 목록보기"** 버튼 보여주기

## 비상 대응 가이드

### 시연 중 문제 발생 시

| 상황 | 대처 |
|------|------|
| AI 감정 분석이 느릴 때 | "AI 모델 로딩에 시간이 걸립니다" 후 미리 분석된 일기 보여주기 |
| 챗봇 응답이 안 나올 때 | "AI 서버와 통신 중입니다" 후 다른 기능 먼저 시연 |
| 음악 추천 크롤링이 오래 걸릴 때 | "YouTube 크롤링 중입니다" 후 미리 검색된 결과 보여주기 |
| 서버 연결 오류 | Spring Boot / FastAPI 서버 상태 확인, 재시작 |
| 로그인 실패 | 다른 테스트 계정 사용 |

### 미리 준비해둘 데이터

1. **작성된 일기** 5-8개 (다양한 감정 + 분석 결과 포함)
2. **챗봇 대화 이력** 2-3건 (Simple/Complex 각각)
3. **의사 계정에 배정된 환자** 3-5명 (일기 데이터 포함)
4. **중증 환자 데이터** 1명 (우울지수 높은 상태)
